## 🚀 Начало работы

### Установка

1.  **Скачайте** [ПОСЛЕДНИЙ РЕЛИЗ](https://github.com/le0booba/Area_Links/releases/latest) с GitHub.
2.  **Распакуйте** ZIP-архив в любую удобную папку на компьютере.
3.  **Откройте Chrome** и перейдите на страницу расширений: `chrome://extensions/`.
4.  **Включите режим разработчика** с помощью переключателя в правом верхнем углу.
5.  **Нажмите «Загрузить распакованное расширение»** и выберите папку с файлами.
6.  **Готово!** Значок Area Links появится на панели инструментов браузера.

### Как пользоваться

| Действие             | Клавиши по умолчанию | Что делать                             |
| ------------------ | ---------------- | ---------------------------------------- |
| **Открыть ссылки**     | `Alt+Z`          | Зажмите клавиши, выделите область мышью, отпустите. |
| **Скопировать ссылки**     | `Alt+X`          | Зажмите клавиши, выделите область мышью, отпустите. |
| **Отменить** | `Esc`            | Нажмите `Esc` во время выделения. |

> **Подсказка:** Горячие клавиши можно изменить на странице `chrome://extensions/shortcuts`.

> **Важно:** Расширение не работает на служебных страницах Chrome (новая вкладка, настройки, страница расширений и т.д.).

## 🌟 Возможности

### Основное
- 🎯 **Визуальное выделение**: Выделяйте ссылки рамкой прямо на странице.
- 🚀 **Два режима работы**: Открывайте ссылки во вкладках или копируйте в буфер обмена.
- ✨ **Подсветка ссылок**: Видно, какие ссылки попали в область — на выбор три цветовые темы (Классический жёлтый, Зелёный маркер, Тёмно-серый).
- 🎨 **Гибкая настройка рамки**: Четыре стиля границы (Сплошная, Пунктир, Точки, Тонкая) и выбор цвета через палитру — плюс четыре ячейки для сохранения любимых цветов.
- 🔗 **Умная фильтрация**: Автоматически пропускает невидимые и нулевые по размеру элементы при сканировании ссылок. Для точной фильтрации по URL используйте исключения доменов и ключевых слов.
- 🖱️ **В контекстном меню**: Активируйте выделение через правую кнопку мыши.
- 📂 **Настройки открытия**: Открывайте в новых вкладках или отдельном окне, размещайте вкладки рядом с текущей или в конце, меняйте порядок открытия.
- ⚙️ **Лимит вкладок**: Ограничьте количество вкладок от 1 до 50 за раз.

### Дополнительно
- 🚫 **Исключения**: Игнорируйте ссылки с определённых доменов или содержащие нужные слова.
- 📥📤 **Импорт и экспорт**: Сохраняйте и восстанавливайте списки исключений или все настройки через JSON-файлы с автоматическим разрешением конфликтов.
- 🧠 **История**: Отдельная история для открытых и скопированных ссылок — до 50 адресов в каждом режиме.
- 🗑️ **Дубликаты**: Убирайте повторы в текущем выделении или предотвращайте повторное открытие уже открытых ссылок.
- 💾 **Синхронизация**: Основные настройки автоматически синхронизируются через аккаунт Chrome.
- 📱 **Быстрый доступ**: Кликните по значку расширения — откроется панель с самыми нужными настройками для режимов открытия и копирования.
- 🌐 **Два языка**: Полная локализация на русском и английском с автоопределением языка системы.
- 🎭 **Разные курсоры**: Для режимов копирования и открытия — сразу понятно, что активно.
- 🎬 **Плавная анимация**: CSS-переходы делают подсветку ссылок приятной для глаз.
- 📋 **Надёжное копирование**: Резервный механизм для копирования работает даже на незащищённых страницах.

## 🔧 Техническая информация

### Оптимизации производительности

1. **Кэширование позиций ссылок**: Координаты всех ссылок вычисляются один раз при нажатии мыши (`mousedown`) и хранятся как обычные числовые значения (`left`, `right`, `top`, `bottom`) в массиве `cachedLinks`. Это полностью исключает обращения к DOM при движении курсора и обеспечивает плавные 60 кадров в секунду даже на страницах с тысячами ссылок. Каждый элемент кэша содержит поле `data: null` для отложенного парсинга URL — hostname и lowercase-варианты вычисляются через `getLinkData()` только в момент первого попадания ссылки в прямоугольник выделения, а не заранее для всех ссылок страницы.

2. **Пакетная отрисовка через RequestAnimationFrame**: События движения мыши обрабатываются через `requestAnimationFrame` с защитой флагом `isUpdateScheduled` в функции `scheduleUpdate()`. Множество быстрых `mousemove`-событий между кадрами сворачиваются в одно обновление экрана, резко снижая нагрузку на процессор. Флаг сбрасывается только после выполнения колбэка кадра — это гарантирует, что лишние вычисления никогда не ставятся в очередь.

3. **Настройки в оперативной памяти**: Объект `settingsCache` в `background.js` объединяет sync и local хранилища в один снимок, загружаемый один раз через `settingsManager.initialize()` при старте. За сессию это сокращает обращения к storage API с потенциально сотен до 1–2. Кэш обновляется точечно через слушатель `chrome.storage.onChanged` — патчатся только изменившиеся ключи, без полной перезагрузки.

4. **История на базе Set (O(1) вместо O(n))**: Массивы истории преобразуются в объекты `Set` (`historySet`, `copyHistorySet`) сразу при инициации выделения в `initSelection()`. Проверки на дубликаты и историю во время движения мыши выполняются через `Set.has()` за константное время вместо линейного перебора массива. При выделении 100 ссылок и истории из 50 адресов это сокращает число операций сравнения с 5 000 до 100 — улучшение в 50 раз, особенно заметное при одновременно включённом обнаружении дубликатов и истории.

5. **Умное переключение классов со статус-трекингом**: Каждая закэшированная ссылка хранит поле `status` с одним из четырёх значений: `0` — вне области, `1` — подсвечена, `2` — дубликат/исключена, `3` — превышен лимит. На каждом кадре изменение класса через `classList.add/remove` происходит только если новый статус отличается от предыдущего. Ссылки, чьё положение относительно области выделения не изменилось, не порождают ни одной DOM-мутации — это минимизирует пересчёт вёрстки и reflow даже при одновременной подсветке сотен ссылок.

### Совместимость
- Использует **Manifest V3** — современный стандарт расширений Chrome
- Работает во всех браузерах на движке Chromium: Chrome, Edge, Brave и других
- Минимальная версия Chrome: 88+
- Соответствует последним требованиям безопасности Chrome

### Умные фишки
- **Автосброс при переключении**: Выделение автоматически отменяется, если переключиться на другую вкладку
- **Фильтрация элементов**: Пропускает невидимые и нулевые по размеру элементы (`ширина ≤ 1px` или `высота ≤ 1px`) при кэшировании ссылок; фильтрация по URL выполняется через исключения доменов и ключевых слов
- **Защита от дубликатов**: Несколько механизмов предотвращения повторов с раздельной историей для каждого режима
- **Международные домены**: Корректно обрабатывает домены на любых языках с автоматической Punycode-конвертацией
- **Запасной вариант**: На незащищённых страницах копирование работает через `document.execCommand`

## 🔒 Разрешения и приватность

Расширение создано с заботой о вашей конфиденциальности.

-   **🚫 Никакого слежения**: Нулевая аналитика, телеметрия отсутствует, данные не собираются.
-   **🔒 Всё локально**: Обработка происходит только в вашем браузере. Никаких сетевых запросов.
-   **🗑️ Полный контроль**: Вы в любой момент можете посмотреть и удалить все данные расширения.

### Что и где хранится

<details>
<summary><strong>🏠 Локальное хранилище (<code>chrome.storage.local</code>)</strong> — только на вашем компьютере, не синхронизируется.</summary>

-   `linkHistory`: Последние 50 открытых ссылок (если включено запоминание).
-   `copyHistory`: Последние 50 скопированных ссылок (если включено запоминание).
-   `excludedDomains`: Список исключённых доменов через запятую.
-   `excludedWords`: Список исключённых слов через запятую.
-   `useHistory`: Включено ли запоминание открытых ссылок.
-   `useCopyHistory`: Включено ли запоминание скопированных ссылок.
</details>

<details>
<summary><strong>☁️ Синхронизируемое хранилище (<code>chrome.storage.sync</code>)</strong> — синхронизируется между всеми вашими браузерами через аккаунт Google.</summary>

-   Основные настройки: `tabLimit`, `selectionBoxStyle`, `selectionBoxColor`, `highlightStyle`, `openInNewWindow`, `openNextToParent`, `reverseOrder`, `applyExclusionsOnCopy`, `removeDuplicatesInSelection`, `checkDuplicatesOnCopy`, `language`, `showContextMenu` и сохранённые цвета (`selectionColorCustomPreset0-3`).
</details>

### Зачем нужны разрешения

<details>
<summary><b>Подробнее о каждом разрешении</b></summary>

| Разрешение         | Для чего                                                                   |
| ------------------ | ------------------------------------------------------------------------- |
| `storage`          | Сохранять ваши настройки и историю ссылок.                                   |
| `tabs`             | Открывать ссылки во вкладках и создавать новые окна.                         |
| `scripting`        | Внедрять на страницы код, который рисует рамку выделения.           |
| `contextMenus`     | Добавлять пункты в меню правой кнопки мыши.   |
| `http://*/*`, `https://*/*` | Работать на любых сайтах, которые вы посещаете.     |

*Расширение запрашивает только необходимый минимум разрешений.*

</details>
